What makes Pathwrite different

The only wizard engine that gives you headless logic, UI shell, and persistence across four frameworks.

🌐

Write once, run in any framework

Your path definition is plain TypeScript. The same guards, validation, and lifecycle hooks work identically in React, Vue, Angular, and Svelte.

🎨

Headless engine + optional UI shell

Use the engine headless for full control, or drop in PathShell for an instant wizard UI with stepper, navigation, and validation display.

💾

Built-in persistence

Save and restore progress with LocalStorage or HTTP. Users pick up where they left off. Configurable strategies, zero boilerplate.

🧪

Test without a browser

Every step, guard, and sub-path is testable by calling engine methods. No DOM, no component mounting.

Define a path in ~30 lines

onboarding.ts 
import type { PathDefinition } from "@daltonr/pathwrite-core";

interface OnboardingData {
  name: string; email: string;
  plan: "free" | "pro" | "enterprise";
}

export const onboarding: PathDefinition<OnboardingData> = {
  id: "onboarding",
  steps: [
    {
      id: "welcome", title: "Your Details",
      canMoveNext: ({ data }) => data.name.trim().length > 0 && data.email.includes("@"),
    },
    {
      id: "plan", title: "Choose Plan",
      canMoveNext: ({ data }) => !!data.plan,
    },
    {
      id: "billing", title: "Billing",
      shouldSkip: ({ data }) => data.plan === "free",  // skip for free plan
    },
    { id: "confirm", title: "Confirm" },
  ],
};

Guards, validation, conditional steps — all declarative, all typed. Works with every adapter.

One path, four adapters

Drop in PathShell for an instant wizard UI in your framework.

CheckoutWizard.tsx 
import { PathShell } from "@daltonr/pathwrite-react";

export default function Checkout() {
  return (
    <PathShell
      path={checkoutPath}
      initialData={{ items: [], total: 0 }}
      onComplete={handleOrder}
      steps={{
        cart:     <CartStep />,
        shipping: <ShippingStep />,
        payment:  <PaymentStep />,
        confirm:  <ConfirmStep />,
      }}
    />
  );
}
CheckoutWizard.vue 
<script setup>
import { PathShell } from "@daltonr/pathwrite-vue";
</script>

<template>
  <PathShell :path="checkoutPath" :initial-data="{ items: [] }"
             @complete="handleOrder">
    <template #cart><CartStep /></template>
    <template #shipping><ShippingStep /></template>
    <template #payment><PaymentStep /></template>
    <template #confirm><ConfirmStep /></template>
  </PathShell>
</template>
checkout.component.html 
<pw-shell
  [path]="checkoutPath"
  [initialData]="{ items: [] }"
  (completed)="handleOrder($event)">

  <ng-template pwStep="cart">
    <app-cart-step />
  </ng-template>
  <ng-template pwStep="shipping">
    <app-shipping-step />
  </ng-template>

</pw-shell>
Checkout.svelte 
<script>
  import { PathShell } from "@daltonr/pathwrite-svelte";
</script>

<PathShell path={checkoutPath} initialData={{ items: [] }}
           oncomplete={handleOrder}>

</PathShell>
