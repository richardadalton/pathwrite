What's New in v0.8.0

Coming soon: Four powerful features that eliminate common boilerplate — zero breaking changes.

🎯

onComplete / onCancel Callbacks

Define completion and cancellation handlers right on your PathDefinition. No more subscribing to events and filtering for type === "completed" — it's where you'd expect it.

onComplete: (data) => submitToAPI(data)
↩️

resetStep() — Undo Current Step

One method to revert the current step's data to what it was when the step was entered. No more manually calling setData for every field.

engine.resetStep(); // data reverts instantly
✨

isDirty — Automatic Dirty Tracking

Know instantly if the user has changed anything on the current step. Perfect for "unsaved changes" prompts or disabling Save buttons. Zero configuration.

snapshot.isDirty // true if data changed
⏱️

stepEnteredAt Timestamp

Captures Date.now() when each step is entered. Compute time spent, trigger timeout warnings, or power analytics — it's just there.

Date.now() - snapshot.stepEnteredAt

All four features are optional, additive, and non-breaking. Ignore them and your code works exactly as before.

One engine. Four idiomatic adapters.

Drop PathShell into any framework and get a full wizard UI — stepper, navigation, validation display and layout — in the same pattern your framework already uses. Define your workflows once and reuse them across React, Vue, Angular and Svelte apps — ideal for multi-framework teams.

CheckoutWizard.tsx 
import { PathShell } from "@daltonr/pathwrite-react";

export default function Checkout() {
  return (
    <PathShell
      path={checkoutPath}
      initialData={{ items: [], total: 0 }}
      onComplete={handleOrder}
      steps={{
        cart:     <CartStep />,
        shipping: <ShippingStep />,
        payment:  <PaymentStep />,
        confirm:  <ConfirmStep />,
      }}
    />
  );
}

Define your path in ~30 lines

Steps, guards, validation, lifecycle hooks — all declarative, all typed.

onboarding.ts 
import type { PathDefinition } from "@daltonr/pathwrite-core";

interface OnboardingData {
  name: string;
  email: string;
  plan: "free" | "pro" | "enterprise";
  billingAddress?: string;
}

export const onboarding: PathDefinition<OnboardingData> = {
  id: "onboarding",
  steps: [
    {
      id: "welcome",
      title: "Your Details",
      canMoveNext: ({ data }) => data.name.trim().length > 0 && data.email.includes("@"),
      validationMessages: ({ data }) => {
        const msgs: string[] = [];
        if (!data.name.trim()) msgs.push("Name is required");
        if (!data.email.includes("@")) msgs.push("Valid email required");
        return msgs;
      },
    },
    {
      id: "plan",
      title: "Choose Plan",
      canMoveNext: ({ data }) => !!data.plan,
    },
    {
      id: "billing",
      title: "Billing",
      // Skip billing for free plan
      shouldSkip: ({ data }) => data.plan === "free",
    },
    {
      id: "confirm",
      title: "Confirm",
    },
  ],
};

That's the entire path definition. Guards, validation, conditional steps — all declarative. Works with every framework adapter.

Persistence that just works.

User closes the tab on step 4? They pick up right where they left off.

App.tsx 
import { restoreOrStart, httpPersistence, HttpStore }
  from "@daltonr/pathwrite-store-http";

const store = new HttpStore({ baseUrl: "/api/wizard" });

const { engine, restored } = await restoreOrStart({
  store,
  key: "user:onboarding",
  path: onboardingPath,
  initialData: { name: "" },
  observers: [
    httpPersistence({
      store,
      key: "user:onboarding",
      strategy: "onNext",  // or "onEveryChange"
    }),
  ],
});

if (restored) {
  showBanner("Welcome back! Resuming where you left off.");
}
Configurable strategies

Save on next, on every data change, or on a custom trigger.

Pluggable stores

Ships with LocalStorageStore and HttpStore. Build your own for IndexedDB or any backend.

Automatic restore

restoreOrStart() checks for saved state first. No conditional logic needed.

Observer pattern

Persistence is just an observer — add analytics, logging, or any side effect the same way.
