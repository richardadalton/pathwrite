The separation that makes everything else possible

Most wizard libraries couple your business logic to UI components. Pathwrite doesn't. It borrows a pattern from programming languages: the clean separation between source code, its interpreter, and the tools that surface its state.

PathDefinition
Source code
Inert, declarative. A plain TypeScript object with steps, guards, and hooks. Does nothing on its own.
PathEngine
Interpreter
Reads the definition and executes it. Manages navigation, enforces guards, fires lifecycle hooks, stacks sub-paths.
PathSnapshot
Program state
Immutable record of position, data, and derived flags at a point in time. Produced after every engine action.
Adapters
Interface
Surface the snapshot to humans using the framework's own reactivity. React hooks, Vue composables, Angular signals, Svelte runes.
Because the definition is inert data, you can package it, version it, test it headlessly, and import it into any adapter — without ever touching UI code.

my-workflows/src/job-application.ts 
// Pure TypeScript — no React, no Vue, no Angular.
// This file is the entire workflow. Ship it as an npm package.
import type { PathDefinition } from "@daltonr/pathwrite-core";

export interface ApplicationServices {
  checkEligibility(years: number): Promise<{ eligible: boolean; reason?: string }>;
  requiresCoverLetter(roleId: string): Promise<boolean>;
}

export function createApplicationPath(svc: ApplicationServices) {
  return {
    id: "job-application",
    steps: [
      { id: "role",    title: "Role",    canMoveNext: ({ data }) => !!data.roleId },
      { id: "experience", title: "Experience" },
      {
        id: "eligibility", title: "Eligibility",
        canMoveNext: async ({ data }) => {
          const r = await svc.checkEligibility(Number(data.yearsExperience));
          return r.eligible ? true : { allowed: false, reason: r.reason! };
        },
      },
      {
        id: "cover-letter", title: "Cover Letter",
        shouldSkip: async ({ data }) => !(await svc.requiresCoverLetter(data.roleId)),
      },
      { id: "review", title: "Review" },
    ],
  } satisfies PathDefinition<ApplicationData>;
}
my-workflows/test/job-application.test.ts 
// No framework. No DOM. No component mounting.
// Test routing, guards, and validation as pure logic.
import { PathEngine } from "@daltonr/pathwrite-core";
import { createApplicationPath } from "../src/job-application";

const fast = {
  checkEligibility: async (y) => y >= 2 ? { eligible: true } : { eligible: false, reason: "Min 2 years required" },
  requiresCoverLetter: async (id) => id === "eng" || id === "data",
};

it("blocks navigation when experience < 2 years", async () => {
  const engine = new PathEngine();
  await engine.start(createApplicationPath(fast), { roleId: "eng", yearsExperience: "1" });
  await engine.next(); // → experience
  await engine.next(); // → eligibility
  await engine.next(); // guard blocks
  expect(engine.snapshot()?.stepId).toBe("eligibility");
  expect(engine.snapshot()?.blockingError).toBeTruthy();
});

it("skips cover letter for non-engineering roles", async () => {
  const engine = new PathEngine();
  await engine.start(createApplicationPath(fast), { roleId: "pm", yearsExperience: "5" });
  await engine.next(); await engine.next(); await engine.next();
  expect(engine.snapshot()?.stepId).toBe("review"); // cover-letter skipped
});

One path, five adapters

Drop in PathShell for an instant wizard UI in your framework. Swap it for usePath() when you want full control.

CheckoutWizard.tsx 
// swap the import for @daltonr/pathwrite-react-native and it just works
import { PathShell } from "@daltonr/pathwrite-react";

export default function Checkout() {
  return (
    <PathShell
      path={checkoutPath}
      initialData={{ items: [], total: 0 }}
      onComplete={handleOrder}
      steps={{
        cart:     <CartStep />,
        shipping: <ShippingStep />,
        payment:  <PaymentStep />,
        confirm:  <ConfirmStep />,
      }}
    />
  );
}
