Live Authwrite engine — runs entirely in the browser, no server.

Built for observable authorization

Authwrite isn't just about allow/deny — it's about understanding why.

🔍

Every decision has a reason

The reason field on every decision carries the ID of the rule that matched — or "default" when none did. No more silent denials.

📝

Pure TypeScript rules

Rules are plain functions — no DSL to learn, no schema to memorise. Your match function receives the full context and returns a boolean.

🔌

Pluggable observers

Wire up OpenTelemetry, Postgres audit logs, Redis rate-limiting, or a custom webhook — all through the same AuthObserver interface.

🚦

Enforcer modes

Switch between enforce, audit (permissive + logged), and lockdown (all denied) without changing a single rule.

🧩

Framework adapters

Drop-in middleware for Express, Fastify, Next.js, and Hono. The core runs in any environment — Node.js, edge runtimes, or the browser.

0️⃣

Zero dependencies

@authwrite/core ships with no runtime dependencies. Framework adapters only depend on their peer framework.

🧪

Testable in isolation

Because your policy is a plain TypeScript object and the engine has no framework coupling, you can test every rule with a simple function call — no HTTP server, no middleware stack, no UI framework to spin up. Pass a subject, resource, and action; assert on the decision. Authorization logic that is this easy to test tends to actually get tested.

The full picture in three files

Define once, evaluate everywhere.

// policy.ts
import { createAuthEngine } from '@authwrite/core'

const engine = createAuthEngine({
  policy: {
    id:            'documents',
    version:       '1.0.0',
    defaultEffect: 'deny',
    rules: [
      {
        id:       'archived-blocks-mutation',
        priority: 20,
        match:    ({ resource }) => resource?.status === 'archived',
        deny:     ['write', 'delete'],
      },
      {
        id:       'admin-full-access',
        priority: 10,
        match:    ({ subject }) => subject.roles.includes('admin'),
        allow:    ['*'],
      },
      {
        id:       'owner-full-access',
        priority: 5,
        match:    ({ subject, resource }) =>
                    !!resource && resource.ownerId === subject.id,
        allow:    ['*'],
      },
    ],
  },
})
// check.ts
const decision = await engine.evaluate({
  subject:  { id: 'user-1', roles: ['editor'] },
  resource: { type: 'document', id: 'doc-42',
               ownerId: 'other-user', status: 'archived' },
  action:   'write',
})

// decision →
// {
//   allowed:    false,
//   reason:     'archived-blocks-mutation',
//   effect:     'deny',
//   defaulted:  false,
//   policy:     'documents',
//   durationMs: 0.12
// }

if (!decision.allowed) {
  logger.warn('access denied', { reason: decision.reason })
}
// middleware.ts
import { createAuthMiddleware } from '@authwrite/express'

app.put('/documents/:id',
  createAuthMiddleware({
    engine,
    subject:  (req) => ({ id: req.user.id, roles: req.user.roles }),
    resource: async (req) => db.documents.get(req.params.id),
    action:   'write',
    onDeny:   (req, res, decision) => {
      res.status(403).json({
        error:  'forbidden',
        reason: decision.reason,   // 'archived-blocks-mutation'
      })
    },
  }),
  updateDocumentHandler
)
