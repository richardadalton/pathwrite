## Practical guidelines

- Open an issue before starting significant work, so we can align on approach.
- Run `npm test` before submitting. All tests must pass.
- Follow the existing code style — TypeScript strict mode, no `any` unless unavoidable.
- Keep commits focused. One logical change per commit.
- Update tests and documentation alongside code changes.

For questions, open a GitHub issue.
